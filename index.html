🌱 Interactive Urban Farming Platform

শহুরে কৃষি প্ল্যাটফর্ম — সম্পূর্ণ বাংলায় ব্যাখ্যা ও ডায়াগ্রাম

📋 প্রজেক্টটা আসলে কী?

এটা একটা শহুরে কৃষি (Urban Farming) প্ল্যাটফর্মের Backend API বানাতে হবে।

সহজ ভাষায় বলতে গেলে — ধরো শহরে যারা থাকে তারা চায় নিজে সবজি/ফল ফলাতে। কিন্তু তাদের জায়গা নেই। আবার শহরের কিছু কৃষকের (farmer) কাছে অতিরিক্ত জায়গা আছে। এই প্ল্যাটফর্ম তাদের connect করে দেবে।

এই প্ল্যাটফর্মে কী কী করা যাবে:

✅ কৃষকদের কাছ থেকে বাগানের জায়গা ভাড়া নেওয়া যাবে

✅ অর্গানিক সবজি/ফল কেনাবেচা করা যাবে (Marketplace)

✅ বাগান করার টিপস শেয়ার করা যাবে (Community Forum)

✅ নিজের গাছের গ্রোথ ট্র্যাক করা যাবে (Real-time)

✅ বিক্রেতাদের অর্গানিক সার্টিফিকেট ভেরিফাই করা হবে

তোমাকে শুধু Backend বানাতে হবে — মানে:

🔹 Database design করতে হবে (Models/Schema)

🔹 REST API বানাতে হবে (Routes + Controllers)

🔹 Authentication system (Login/Register with JWT)

🔹 Role-based access control (কে কী করতে পারবে)

🔹 Rate limiting, Error handling, Pagination

🔹 API documentation (Swagger/Postman)

🛠️ কোন Technology ব্যবহার করতে পারো?

Assignment-এ specific বলা নেই, তবে Junior Backend Developer হিসেবে সবচেয়ে ভালো হবে:

Node.js + Express.js — সবচেয়ে সহজ ও জনপ্রিয়

MongoDB + Mongoose — NoSQL database, সহজে শেখা যায়

অথবা PostgreSQL + Prisma/Sequelize — SQL চাইলে

JWT — Authentication এর জন্য

Socket.io — Real-time plant tracking এর জন্য

Swagger — API documentation এর জন্য

⚡ মূল ফিচার সমূহ (Core Modules)

🔐

Authentication ও Role-Based Access

ইউজার Register/Login করবে। JWT token দিয়ে authenticate হবে। ৩টা role: Admin, Vendor, Customer — প্রত্যেকের আলাদা permission।

🌿

বাগানের জায়গা ভাড়া (Garden Rental)

Vendor তার জায়গা list করবে (location, size, price)। Customer সেটা search করবে ও booking করবে। Location-based search থাকবে।

🛒

মার্কেটপ্লেস (Produce Marketplace)

Vendor সবজি/ফল/বীজ/টুলস লিস্ট করবে। Customer কিনতে পারবে। Pagination থাকবে, category-wise filter থাকবে।

📡

Real-time Plant Tracking

গাছের growth, health, harvesting time ট্র্যাক করা। Socket.io বা similar tech দিয়ে real-time update দেওয়া।

🏅

Sustainability Certification

Vendor কে organic সার্টিফিকেট দিতে হবে। Admin verify করবে। Verify ছাড়া product list হবে না।

💬

Community Forum

ইউজাররা বাগান করার টিপস, eco-friendly practices শেয়ার করবে। পোস্ট করা, পড়া — সবাই পারবে।

⚠️

Error Handling ও Rate Limiting

সব API-তে consistent JSON response। HTTP status codes ঠিকমতো ব্যবহার। Login/Register route-এ rate limiting।

📄

API Documentation

Swagger/OpenAPI অথবা Postman Collection দিতে হবে। সব API-র sample request ও response সহ।

👥 ৩টা Role আছে এই সিস্টেমে

🛡️ Admin (প্ল্যাটফর্মের মালিক)

Admin সবকিছু control করে। সে পারবে:

• সব ইউজার manage করতে (block/unblock)

• Vendor দের approve করতে

• Organic certification verify করতে

• Platform activity monitor করতে

• Product accuracy check করতে

🧑‍🌾 Vendor / Urban Farmer (কৃষক)

Vendor হলো যে চাষ করে। সে পারবে:

• নিজের প্রোফাইল ও ফার্ম register করতে

• বাগানের জায়গা ভাড়ায় দিতে (Rental Space)

• সবজি/ফল list করতে (Produce)

• অর্ডার manage করতে

• Organic সার্টিফিকেট submit করতে

🛍️ Customer (সাধারণ ইউজার)

Customer হলো যে কিনবে বা ভাড়া নেবে। সে পারবে:

• বাগানের জায়গা ভাড়া নিতে

• সবজি/ফল কিনতে (Order)

• নিজের গাছ track করতে

• Community forum-এ পোস্ট করতে

• রিভিউ দিতে

🗄️ ডাটাবেজ মডেল (৭টা টেবিল/Collection)

নিচে প্রতিটা মডেলের ফিল্ড, টাইপ, এবং বাংলায় কী কাজ করে — সব দেওয়া আছে:

👤 User (ইউজার)
🔑 idObjectId / UUID
nameString — নাম
emailString (unique) — ইমেইল
passwordString (hashed) — পাসওয়ার্ড
roleEnum: admin/vendor/customer
statusEnum: active/blocked — একাউন্ট স্ট্যাটাস
createdAtDate — কবে তৈরি হয়েছে
🧑‍🌾 VendorProfile (বিক্রেতা প্রোফাইল)
🔑 idObjectId / UUID
🔗 userId→ User.id — কোন ইউজারের
farmNameString — ফার্মের নাম
certificationStatusEnum: pending/approved/rejected
farmLocationString — ফার্মের ঠিকানা
🥬 Produce (পণ্য/ফসল)
🔑 idObjectId / UUID
🔗 vendorId→ User.id — কোন বিক্রেতার
nameString — পণ্যের নাম
descriptionString — বিবরণ
priceNumber — দাম
categoryString — ক্যাটাগরি (সবজি/ফল/বীজ)
certificationStatusEnum — অর্গানিক ভেরিফাইড কিনা
availableQuantityNumber — কতটুকু আছে
🏡 RentalSpace (ভাড়ার জায়গা)
🔑 idObjectId / UUID
🔗 vendorId→ User.id — কোন কৃষকের
locationString — কোথায় আছে
sizeString/Number — কত বড়
priceNumber — ভাড়া কত
availabilityBoolean — খালি আছে কিনা
📦 Order (অর্ডার)
🔑 idObjectId / UUID
🔗 userId→ User.id — কে অর্ডার দিয়েছে
🔗 produceId→ Produce.id — কোন পণ্য
🔗 vendorId→ User.id — কোন বিক্রেতার কাছ থেকে
statusEnum: pending/confirmed/delivered/cancelled
orderDateDate — অর্ডারের তারিখ
💬 CommunityPost (কমিউনিটি পোস্ট)
🔑 idObjectId / UUID
🔗 userId→ User.id — কে পোস্ট করেছে
postContentString — পোস্টের কন্টেন্ট
postDateDate — কবে পোস্ট হয়েছে
🏅 SustainabilityCert (সার্টিফিকেট)
🔑 idObjectId / UUID
🔗 vendorId→ User.id — কোন বিক্রেতার
certifyingAgencyString — কোন সংস্থা দিয়েছে
certificationDateDate — কবে পেয়েছে

🔗 মডেলগুলোর মধ্যে সম্পর্ক (Relationships)

কোন টেবিল কার সাথে কিভাবে connected — সেটা বোঝা খুব জরুরি:

সম্পর্কধরনবাংলায় ব্যাখ্যা
User → VendorProfile One-to-One একজন User-এর একটাই VendorProfile থাকবে (যদি সে Vendor হয়)
User → Produce One-to-Many একজন Vendor অনেক পণ্য (Produce) list করতে পারবে
User → RentalSpace One-to-Many একজন Vendor অনেকগুলো ভাড়ার জায়গা দিতে পারবে
User → Order One-to-Many একজন Customer অনেকগুলো অর্ডার দিতে পারবে
Produce → Order One-to-Many একটা পণ্যের বিপরীতে অনেক অর্ডার আসতে পারে
User → CommunityPost One-to-Many একজন ইউজার অনেক পোস্ট করতে পারবে
User → SustainabilityCert One-to-Many একজন Vendor-এর একাধিক সার্টিফিকেট থাকতে পারে

🔌 তোমাকে যে API Routes বানাতে হবে

🔐 Auth Routes (/api/auth)

POST/api/auth/registerনতুন একাউন্ট তৈরি
POST/api/auth/loginলগইন করে JWT token পাওয়া

👤 User Routes (/api/users) — Admin Only

GET/api/usersসব ইউজার দেখা
GET/api/users/:idএকজন ইউজারের ডিটেইল
PUT/api/users/:id/statusইউজার block/unblock

🥬 Produce Routes (/api/produce)

GET/api/produceসব পণ্য দেখা (pagination + filter)
GET/api/produce/:idএকটা পণ্যের ডিটেইল
POST/api/produceনতুন পণ্য add (Vendor only)
PUT/api/produce/:idপণ্য update (Vendor only)
DELETE/api/produce/:idপণ্য delete (Vendor/Admin)

🏡 Rental Routes (/api/rentals)

GET/api/rentalsসব জায়গা দেখা (location search)
POST/api/rentalsজায়গা add করা (Vendor)
POST/api/rentals/:id/bookজায়গা ভাড়া নেওয়া (Customer)

📦 Order Routes (/api/orders)

POST/api/ordersঅর্ডার দেওয়া (Customer)
GET/api/ordersনিজের অর্ডার দেখা (pagination)
PUT/api/orders/:id/statusঅর্ডার status update (Vendor)

💬 Community Routes (/api/community)

GET/api/communityসব পোস্ট দেখা
POST/api/communityনতুন পোস্ট করা (logged in user)
DELETE/api/community/:idপোস্ট delete (own/Admin)

🏅 Certification Routes (/api/certifications)

POST/api/certificationsসার্টিফিকেট submit (Vendor)
GET/api/certificationsসব সার্টিফিকেট দেখা (Admin)
PUT/api/certifications/:id/verifyverify করা (Admin only)
